# REPO SCOPE

## 这个公开仓库包含什么

- `index.html`：对外产品入口页
- `README.md`：产品简介与仓库定位
- `STATUS.md`：当前阶段与下一步
- `MARKET_LOOP.md`：市场切口、扩张路径与沟通闭环
- `SCENARIOS.md`：第一阶段最重要的使用场景
- `assets/screenshots/`：精选视觉预览
- `assets/poster/`：一页纸 / 海报类材料

## 这个公开仓库不包含什么

以下内容默认不公开：
- 完整开发源
- 内部产品规格、records、运行轨迹与历史判断材料
- 提示词、规则链、池规则、评估链
- 能直接反推核心方法论与实现路径的细节
- 私有运行环境、数据结构与执行主链

## 为什么这样分层

未竟之意真正的价值不只是某一个页面、某一个模型能力或某一个自动化动作，
而是从“方向形成”到“持续推进”的完整结构。

公开仓库负责让外部看见这条结构已经长出来了；
更深的实现、验证和判断层，只在下一层沟通里展开。

## 这不是在隐藏空白

这层分工不是为了把空白藏起来，而是为了避免把尚未对外开放、也不适合直接暴露的核心资产平铺在公开仓里。

对外仓要承担的是第一层说服力，不是全部真相源。
